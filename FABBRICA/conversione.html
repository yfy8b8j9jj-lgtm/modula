Conversione & migrazione

Il cliente arriva da un altro gestionale? Qui i suoi file (Excel, CSV, PDF, foto) vengono convertiti nel formato di Modula ed entrano nella sua app con un clic. Il motore è la skill importa-dati.

Come funziona
1 · ENTRATA
File altro gestionale
Excel/CSV, PDF (anche scansioni), foto. Ogni gestionale ha il suo tracciato.
→
2 · SKILL
importa-dati normalizza
Allinea allo schema Modula (core.js MAPS), pulisce date/indirizzi/importi.
→
3 · USCITA
modula-import.json
File pulito, con anteprima in chat: conteggi, righe campione, avvisi.
→
4 · APP
📥 Importa dati
Si carica nell'app del cliente: anteprima in-app → Conferma → dati dentro.
Cosa converte — le 4 entità
👥 Clienti
firstName · lastName · street · streetNo · cap · town · phone · email · group · notes
🪵 Pellet / consegne
clientName · qty · unit (sacchi/t) · date · price · status · notes
🔧 Manutenzioni
clientName · title · date · type · price · status · notes
📅 Appuntamenti
clientName · title · date · time · done

Importa solo le entità dei moduli che il cliente ha attivato. Fonte di verità dei campi: const MAPS in core/core.js.

Privacy — sempre
🔒 I file di un cliente vero contengono dati personali (GDPR): si lavora solo in locale, niente upload esterni, e mai nel repo pubblico. Il Convertitore lavora in coppia con la skill dati-sensibili. Nessun dato inventato: campo assente → vuoto.
Gestionali mappati
Carico…

Ogni gestionale nuovo lascia un tracciato riutilizzabile in FABBRICA/agenti/convertitore/MEMORIA.md: la volta dopo, stessa fonte → conversione immediata.

Per avviare: dai i file a Claude e di' «converti questi dati dal gestionale X» — oppure invoca il subagent fabbrica-convertitore. Parte la skill importa-dati.
